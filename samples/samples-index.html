dart-squid Sample #1

Sample #1 (SVG wrapped in HTML)

This is a very basic visual demonstration of the dart-squid Widgets in action. Play around a bit and you will notice some alignment features (between widgets and related to browser-window boundaries), some position-constraint features that invoke during movement, the ability to hide/show Widgets (including any hierarchically contained), multi-select, re-size/movement, and so forth.

Sample #1 (SVG)

Note: Until reported Dart Issue 2977 (ElementImplementation behavior fluctuates based on containing doc type (HTML vs. SVG) is resolved, this example will not fully work (it may even "aw-snap" the first time; refresh browser and it should partially work (showing widgets will move/size, sorta) and allow you to see where innerHTML issue is hit that crashes things. The same Dart code, run from both a standalone SVG document and from within an HTML document, will not work in the SVG document due to weird object-hierarchy issues or something that makes "innerHTML" unknown. Notice that the HTML version of this example (above) is just a wrapper around the identical SVG structure.

Another issue with the non-wrapped SVG version is odd behavior with event-code removal, thus making the selection (and de-selection) of widgets unpredictable or impossible. Again, this may be very much related to issue 2977 (with regards to the element hierarchy in DART) since, by simply wrapping this SVG doc with a dummy-HTML file, the behavior within the embedded SVG is totally as expected and predictable. Clearly something is screwy in the Dart HTML library with regards to SVG elements that are "standalone" vs. "wrapped".

The ultimate goal was to remove the need for an outer HTML "wrapper" and have the entire "application" be just an SVG document with all sorts of embedded functionality and interactivity. This does work (these widgets were first implemented in JS prior to Dart's existence and it worked fine there).
